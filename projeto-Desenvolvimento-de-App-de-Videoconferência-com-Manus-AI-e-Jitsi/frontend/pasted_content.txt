# Documentação - Sistema de Videoconferência Jitsi Meet

## Visão Geral

O ImóvelConnect integra o **Jitsi Meet** para videoconferências em tempo real entre clientes e corretores de imóveis. O sistema foi otimizado para funcionar perfeitamente em dispositivos móveis Android e iOS.

## Arquitetura

### Componentes Principais

| Componente | Localização | Responsabilidade |
|-----------|------------|------------------|
| **VideoCall Screen** | `app/videocall/[roomId].tsx` | Interface de videochamada com Jitsi |
| **Chat Integration** | `app/chat/[id].tsx` | Iniciar videochamada do chat |
| **Broker Profile** | `app/broker/[id].tsx` | Iniciar videochamada com corretor |
| **Property Detail** | `app/property/[id].tsx` | Iniciar videochamada sobre propriedade |
| **Appointment API** | `server/routers.ts` | Criar agendamentos com jitsiRoomId |
| **Database** | `drizzle/schema.ts` | Armazenar dados de agendamentos |

### Fluxo de Dados

```
Usuário clica em "Iniciar Videochamada"
    ↓
Validação de autenticação e dados
    ↓
Criar agendamento com jitsiRoomId único
    ↓
Navegar para tela de videochamada
    ↓
Carregar Jitsi Meet com configurações
    ↓
Usuários se conectam à sala
    ↓
Videochamada ativa
    ↓
Usuário encerra ou sai
    ↓
Atualizar status do agendamento
```

## Configuração do Jitsi

### Servidor Público

O aplicativo usa o servidor público **meet.jit.si** (sem autenticação necessária):

```javascript
const domain = 'meet.jit.si';
```

## Geração de Room ID

### Formato

```javascript
const jitsiRoomId = `imovelconnect-${Date.now()}-${Math.random().toString(36).substr(2, 9)}`;
```

**Exemplo**: `imovelconnect-1715425200000-abc123def45`

### Características

- **Único**: Combinação de timestamp + string aleatória garante unicidade
- **Legível**: Prefixo "imovelconnect" identifica a origem
- **Seguro**: Não sequencial, difícil de adivinhar
- **Persistente**: Armazenado no banco de dados para referência futura

## Tratamento de Erros

### Tipos de Erro

| Erro | Causa | Solução |
|------|-------|---------|
| `connectionFailed` | Sem internet ou servidor indisponível | Verificar conexão, tentar novamente |
| `connectionInterrupted` | Queda de conexão durante chamada | Reconectar automaticamente |
| `errorOccurred` | Erro genérico do Jitsi | Verificar logs, tentar novamente |
| `timeout` | Demora > 15s para conectar | Verificar banda, tentar novamente |
| `UNAUTHORIZED` | Usuário não autenticado | Fazer login |

### Retry Automático

O sistema permite até **3 tentativas** de reconexão:

```javascript
if (retryCount < 3) {
  setError(false);
  setLoading(true);
  setRetryCount(retryCount + 1);
  // Tentar novamente
}
```

### Timeouts

| Timeout | Duração | Propósito |
|---------|---------|----------|
| Load | 15s | Tempo máximo para carregar Jitsi |
| Fallback | 10s | Esconder loading se Jitsi não responder |

## Troubleshooting

### Problema: Jitsi não carrega

**Causa**: Servidor meet.jit.si indisponível ou sem internet

**Solução**:
1. Verificar conexão com internet
2. Aguardar 15 segundos
3. Clicar "Tentar novamente"
4. Se persistir, verificar status de meet.jit.si

### Problema: Áudio/Vídeo não funciona

**Causa**: Permissões não concedidas ou dispositivo sem câmera/microfone

**Solução**:
1. Verificar permissões do app (Configurações > Permissões)
2. Conceder acesso a câmera e microfone
3. Reiniciar app
4. Tentar novamente

### Problema: Conexão cai durante chamada

**Causa**: Internet fraca ou instável

**Solução**:
1. Verificar sinal de internet
2. Aproximar-se do roteador WiFi
3. Desativar outras apps usando internet
4. Tentar novamente com conexão melhor

### Problema: Timeout ao conectar

**Causa**: Conexão muito lenta

**Solução**:
1. Verificar velocidade de internet
2. Usar WiFi em vez de dados móveis
3. Fechar outros apps
4. Tentar novamente

## Performance

### Otimizações Implementadas

1. **Resolução Adaptativa**: 720p ideal, ajusta para 360p-1080p conforme banda
2. **P2P Habilitado**: Conexão direta entre participantes quando possível
3. **Layer Suspension**: Suspende camadas de vídeo em conexões lentas
4. **Opus Red**: Redundância de áudio para melhor qualidade
5. **Noise Detection**: Detecta e avisa sobre ruído de microfone

### Requisitos de Banda

| Qualidade | Upload | Download | Recomendado |
|-----------|--------|----------|-------------|
| 360p | 500 kbps | 500 kbps | Conexão fraca |
| 720p | 2.5 mbps | 2.5 mbps | Conexão normal |
| 1080p | 5 mbps | 5 mbps | Conexão forte |

## Segurança

### Considerações

1. **Sem Autenticação Jitsi**: Meet.jit.si é público, qualquer pessoa com room ID pode entrar
2. **Room ID Único**: Difícil adivinhar, mas não impossível
3. **Sem Criptografia End-to-End**: Jitsi Meet não oferece E2EE por padrão

### Recomendações

1. Usar IDs únicos e aleatórios (já implementado)
2. Não compartilhar room ID em canais públicos
3. Encerrar sala após uso
4. Usar HTTPS para todas as conexões
5. Considerar Jitsi Meet self-hosted para produção crítica
